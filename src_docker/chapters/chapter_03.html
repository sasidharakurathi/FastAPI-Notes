The Famous "Hello World"

Let's run our very first container. We open our terminal and type exactly this:

docker run hello-world

Here is what happens behind the scenes in milliseconds:

  1. The Docker Client asks our local daemon to run the `hello-world` image.
  2. The daemon checks our laptop's local cache. It initially does not find it.
  3. The daemon automatically connects to Docker Hub (the cloud registry) and natively pulls the image down.
  4. The daemon sparks up the container, runs the tiny "hello" code inside it, streams the output to our terminal, and immediately stops the container.

Interactive Containers (-it)

Running a single command is cool, but what if we just want to "be" inside a Linux machine? This is where the interactive flags come in.

docker run -it ubuntu bash
What's happening here?

-i (Interactive): Tells Docker to keep the STDIN pipe open so we can type.
-t (TTY): Allocates a pseudo-TTY (gives us a working terminal cursor).
ubuntu: The exact image we are booting.
bash: The command we want to run the second the container wakes up.

Our terminal prompt will instantly change. We are now the `root` user inside an isolated, bare-bones Ubuntu container on our machine!

Lifecycle Commands: ps, stop, start, rm

To control the thousands of containers we will eventually build, we need specific daemon commands.

docker ps
Lists only the containers that are actively running right now. (Similar to the Linux `ps` command).

docker ps -a
Lists ALL containers, including stopped, dead, or crashed ones. 
# Gracefully stop a running container
docker stop <container_id>

# Wake up a previously stopped container
docker start <container_id>

# Completely delete the container off our hard drive
docker rm <container_id>
⚠️ Warning: Orphaned Containers!
Every time we type `docker run`, Docker creates a brand new container instance. If we run a container 10 times, we spin up 10 containers. When they stop, they sit there dead on our hard drive, slowly eating up gigabytes of space. We must actively `docker rm` them or use the `--rm` flag (below).

The 6 Sacred Flags of `docker run`

Every developer memorizes these six flags. We combine them to shape exactly how our container behaves.

Flag Name Plain English Meaning
-d Detached Runs the container secretly in the background so it doesn't lock up our terminal cursor.
-p 8080:80 Publish (Ports) "Map port 8080 on our laptop, straight through the wall, to port 80 inside the container."
-v /home:/app Volume "Mount our laptop's `/home` folder directly into the container's `/app` folder." (Live syncs files).
-e KEY=val Environment Injects a secret environment variable (like a Database URL or API Key) into the container on boot.
--name my_app Name Gives the container a human-readable name so we don't have to target random IDs like `a8e914023x`.
--rm Remove Auto-destruct! The exact second the container stops, instantly delete it to save disk space.

Real Example: Running Nginx Locally

Let's immediately apply the six sacred flags to run something highly practical. Nginx is the world's most popular web server. Let's host it in 5 seconds without installing anything on our host machine.

docker run --rm -d --name local-web -p 8080:80 nginx

Let's map out exactly what we just commanded the daemon to do:

  • --rm: When we kill this web server later, permanently delete the container instance.
  • -d: Run the web server in the background so we can keep typing in this terminal tab.
  • --name local-web: Call it 'local-web' so it's easy to stop later.
  • -p 8080:80: If we go to localhost:8080 on our laptop's browser, forward our traffic right into port 80 of the Nginx container.
  • nginx: Download the official Nginx image from the cloud.

We open our normal web browser and navigate to http://localhost:8080.
We will instantly see the "Welcome to nginx!" splash page.

When we are done testing our proxy logic, we tear down the container instantly by typing:

docker stop local-web

Because we used the --rm flag earlier, stopping it also automatically triggered its deletion. Zero mess left on our laptop!
